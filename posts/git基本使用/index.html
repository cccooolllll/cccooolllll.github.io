1 前提

git是一款免费、开源的分布式版本控制系统；

github是使用git做版本控制的代码托管平台。

git可以本地使用，github不是必需品。

如果想要使用一个网络服务托管自己不同版本的代码，那么可以使用github、gitlab或者gitee等使用git的代码托管平台。

1.1 Git 有四个区域:

1.1.1 3个本地区域

  • 工作区(Workspace): 存放项目代码的地方。
  • 暂存区(Stage): 存放临时的改动，事实上它只是一个文件，保存即将提交的文件列表信息。
  • 资源库(Repository): 安全存放数据的位置，这里面有提交到所有版本的数据。其中 HEAD 指向最新放入仓库的版本。

1.1.2 1个远程区域

  • 远程库(Remote): 远程托管代码的服务器，比如github。

2 Git初始化

1

$ git init

初始化成功后，会出现一个.git的隐藏文件夹，这个就是项目的git仓库。

3 Git配置

3.1 用户配置-git config

查看完整的git配置列表，其中需要手动配置的是user.name和user.email。

1

$ git config --list

或者也可以在.git文件夹里中的config文件中查看。

3.1.1 配置命令范围：

1
2

$ git config		  # 仅对当前仓库
$ git config --global # 作用于系统上的所有仓库

3.1.2 设置用户信息：

必须配置这两个信息。

1
2
3
4
5

# 设置用户名，去掉--global为仅对当前仓库生效
$ git config --global user.name "UserName"	

# 设置邮箱，去掉--global为仅对当前仓库生效
$ git config --global user.email "Email"

3.1.3 查看用户信息：

当你不记得这个仓库的用户信息时，可以尝试使用该命令查看。还可以使用git log来查看之前发起过提交的用户信息，这个下面会提到。

1
2

$ git config user.name          # 查看该git的用户名
$ git config user.email         # 查看该git的email

3.1.4 添加远程仓库：

当你要将文件推送到Github或者Gitee等代码托管平台时，需要添加远程仓库链接。

1

$ git remote add origin [远程仓库git链接]

4 Git常用命令

对我来说，Git最基本的命令是add、commit、status。

  • git status : 查看文件夹中的文件，自从上次提交过之后，被修改的文件
  • git add : 将文件添加到暂存区
  • git commit : 提交暂存区文件到本地仓库
  • git rm : 删除文件
  • git diff : 比较文件在暂存区和工作区的差异
  • git log : 展示git提交记录，可以查看每次提交的用户信息以及提交信息

4.1 git status

查看当前文件夹git状态。如果自上次提交过之后，文件夹内的文件有过修改，该命令会列出修改过的文件，并提示接下来应该怎么做，比如add或者commit。 如果文件夹中所有文件都提交commit到本地资源库中，该命令会提示：Nothing to commit。

$ git status -s		# 查看详细文件状态

4.2 git add

添加文件到暂存区。

1
2
3
4
5

# 1. 添加单个文件到暂存区，文件<file>必须包含扩展名
$ git add <file>    

# 2. 添加文件夹中所有内容到暂存区，一个点"."，指当前文件夹
$ git add .

4.3 git commit

提交暂存区文件到本地资源仓库。

1
2
3
4
5

$ git commit
# 输入这个指令之后，会跳转到一个vim编辑器界面，需要在这个界面中编辑提交的信息，适合输入长信息。

# 如果在指令之后添加 -m ，只需要在 -m 之后输入提交的信息就可以了，适合简短信息。
$ git commit -m [message]

提交暂存区的指定文件到本地仓库。

1

$ git commit [file1] [file2] ... -m [message]

4.4 git rm

1、将文件从暂存区和工作区中删除

1

$ git rm README.md

2、将文件从暂存区和工作区中强制删除

1
2

# 可以加上 -f，表示强制删除之前修改过而且 add 到暂存区的文件
$ git rm -f README.md

3、将文件从暂存区删除，在工作区保留

1

$ git rm --cached README.md

4.5 git diff

比较文件的不同，即比较文件在暂存区和工作区的差异。 git diff 显示已经写入暂存区和已经被修改但尚未写入暂存区文件的区别。

应用场景:

# 查看工作区和暂存区相同文件的差异，并输出差异内容
$ git diff		

# 查看已经缓存的改动
$ git diff --cached		

# 查看工作区和暂存区所有相同文件的差异，并输出差异内容
$ git diff HEAD	

# 显示差异的摘要，不输出差异内容
$ git diff --stat

4.9 .gitignore

git忽略文件。 在文件夹中添加一个名为.gitignore的文件，文件里可以添加文字数据。在该文件中添加的文件名，git会忽略掉。 举例：

1
2
3

Pic.png
*.txt
Temp/

在.gitignore中添加这三行内容。 第一行表示，git会忽略掉所有名为Pic.png的文件。 第二行表示，git会忽略掉所有以.txt结尾的文件。 第三行表示，git会忽略掉所有名为Temp的文件夹。

被git忽略的文件的修改或删除，git将不会再有任何提示。
